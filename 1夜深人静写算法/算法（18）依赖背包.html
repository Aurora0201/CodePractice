二、简单依赖背包问题

  • 这章讲完，背包问题也就告一段落了，作者会在下一章对之前讲到的 0/1 背包、完全背包、多重背包、分组背包 以及今天要讲的依赖背包进行一个全面的总结，求同存异，希望读者看完以后，能够对动态规划有一个更加深入的理解。那么，我还是通过一个简单的例题来描述下什么是依赖背包。

【例题1】商店里有 n(n≤50)n(n \le 50)n(n≤50) 个盒子，每个盒子价钱为 p[i](p[i]≤1000)p[i](p[i] \le 1000)p[i](p[i]≤1000)，价值为 0，盒子里面有一些小礼物，数量为 m[i](1≤m[i]≤10)m[i](1 \le m[i] \le 10)m[i](1≤m[i]≤10)，每个小礼物描述成一个二元组 (c,v)(c, v)(c,v)，ccc 为价钱，vvv 为价值，如果要买小礼物，必须先买盒子。现在给出价钱 m(m≤100000)m(m \le 100000)m(m≤100000)，求能够买到的最大价值。

  • 这是一个比较特殊的依赖性背包问题，也是依赖背包中最简单的情况，其中盒子作为 主件，小礼物作为 附件。想要购买附件，必须先购买主件，此所谓 “依赖” ；

1、状态设计

  • 第一步：设计状态；
  • 状态 (i,j)(i, j)(i,j) 表示前 iii 个盒子购买的价钱恰好为 jjj (i∈[0,n],j∈[0,m])(i \in [0, n], j \in [0, m])(i∈[0,n],j∈[0,m])；
  • 令 dp[i][j]dp[i][j]dp[i][j] 表示状态 (i,j)(i, j)(i,j) 下得到的最大价值，即前 iii 个盒子购买价钱为 jjj 的情况下所得到的最大总价值；

2、状态过程设计

  • 我们在设计状态的时候，没有把小礼物设计到状态里，那么如何进行状态转移呢？
    图二-2-1
  • 可以这么考虑，抛开小礼物不说，每个盒子其实只有两种状态，选 和 不选；
  • 1）选：就是要对这个盒子里的小礼物进行一次 0/1 背包；
  • 2）不选：就和这盒子里的小礼物无关了，直接等于前 i−1i-1i−1 个盒子的最大价值，即 dp[i−1][j]dp[i-1][j]dp[i−1][j]。
  • 那么，只要从前往后枚举所有盒子，p[i]p[i]p[i] 为第 iii 个盒子的价钱，w[i]w[i]w[i] 为价值，由于这个问题下盒子是没有价值的，即 w[i]w[i]w[i] 恒等于零；

进行如下三步操作：
  1）首先，买第 iii 个盒子，并且不放物品；
  2）然后，既然盒子都已经买下来了，就可以放心对第 iii 个盒子里的小礼物进行 0/1 背包了；
  3）最后，对 买盒子 iii 和 不买盒子 iii 取一次最大价值；

3、算法详解

1）买盒子

  • 买第 iii 个盒子，不放物品的情况肯定是从前 i−1i-1i−1 个盒子的状态推算过来的，给出状态转移方程如下：
    dp[i][j]={infj<p[i](1)dp[i−1][j−p[i]]+w[i]j≥p[i](2)dp[i][j] =\begin{cases}inf & j < p[i] & (1)\\dp[i-1][j - p[i]]+w[i] & j \ge p[i] & (2)\end{cases}dp[i][j]={infdp[i−1][j−p[i]]+w[i]​j<p[i]j≥p[i]​(1)(2)​
  • (1) 代表钱不够，无限凄凉；(2) 代表从 前 i−1i-1i−1 个盒子价格为 j−p[i]j - p[i]j−p[i] 时，再花 p[i]p[i]p[i] 买下第 iii 个盒子的情况。这时候 dp[i][j]dp[i][j]dp[i][j] 代表的是 第 iii 个盒子买下后，还没买小礼物时，容量为 jjj 的最大总价值；

2）买小礼物

  • 既然盒子都已经买下来了，就可以放心对第 iii 个盒子里的小礼物在 dp[i][0...m]dp[i][0...m]dp[i][0...m] 上进行 0/1 背包了；所有小礼物都枚举完毕以后，得到的 dp[i][j]dp[i][j]dp[i][j] 代表的是 第 iii 个盒子买下，并且买了若干小礼物后，容量为 jjj 的最大总价值；

3）抉择

  • 最后，对买 这个盒子 和 不买这个盒子 做一次选择，即取一次最大价值，如下：dp[i][j]=max(dp[i][j],dp[i−1][j])dp[i][j] = max(dp[i][j], dp[i-1][j])dp[i][j]=max(dp[i][j],dp[i−1][j])
  • 这里的 dp[i−1][j]dp[i-1][j]dp[i−1][j] 正代表的是第 iii 个盒子不买的情况；

三、一般依赖背包问题

  • 接下来，我们来看看更加一般情况下的依赖背包问题。

1、问题引入

【例题2】有 n(n≤100)n(n \le 100)n(n≤100) 个物品和一个容量为 m(m≤100)m(m \le 100)m(m≤100) 的背包。物品之间具有依赖关系，且依赖关系组成一棵树的形状。如果选择一个物品，则必须选择它的父结点。 第 iii 件物品的容量是 c[i]c[i]c[i]，价值是 w[i]w[i]w[i]，依赖的父结点编号是 p[i]p[i]p[i]。求解将哪些物品装入背包，可使物品总容量不超过背包容量，且总价值最大。

  • 这个问题将【例题1】的 “简单依赖” 推广到了更加一般的情况，看似复杂了不少。
  • 然而， 任何复杂的问题都是由一些简单的问题组成的，动态规划就是一个最好的例子，如何把它的子问题抽象出来就是问题的关键。
  • 对于这个问题，我们把每个结点为根的子树作为一个子问题，以这个为突破口来寻找答案。

2、设计状态

  • 更加具体的，我们令 dp[i][j]dp[i][j]dp[i][j] 表示以 iii 为根的子树，在容量 jjj 的情况下获得的最大价值。所以 dp[root][m]dp[root][m]dp[root][m] 就是我们要求的答案（其中 rootrootroot 为物品依赖树的根结点，mmm 为例题中提到的背包容量）。
  • 那么，在某个子树的根结点为 uuu，它的某个子结点为 vvv 的情况下，我们可以通过递归的方式计算出 dp[v][i](i∈[0,m])dp[v][i] (i \in [0, m])dp[v][i](i∈[0,m])，然后把 iii 理解成物品容量，dp[v][i]dp[v][i]dp[v][i] 理解成物品价值，得到一个最多 m+1m+1m+1 个物品的物品组：(0，dp[v][0]),(1,dp[v][1])...(m,dp[v][m])(0，dp[v][0]), (1, dp[v][1]) ... (m, dp[v][m])(0，dp[v][0]),(1,dp[v][1])...(m,dp[v][m])
  • 然后通过遍历子树的所有子结点完毕后，进行一次分组背包计算出所有 dp[u][i](i∈[0,m])dp[u][i] (i \in [0, m])dp[u][i](i∈[0,m]) 的值，假设 uuu 有 kkk 个子结点，则对于 uuu 来说，一共有 kkk 个物品组。

3、状态转移方程

  • 令 某个子树为 uuu，它的某个子结点为 vvv，则有状态转移方程如下：dp[u][i]=max⁡v∈son(u)(dp[u][i],dp[u][i−j]+dp[v][j])dp[u][i] = \max_{v \in son(u)}(dp[u][i], dp[u][i-j] + dp[v][j])dp[u][i]=v∈son(u)max​(dp[u][i],dp[u][i−j]+dp[v][j]) i∈[0,m−c[u]],j∈[0,i]i \in [0,m-c[u]], j \in [0, i]i∈[0,m−c[u]],j∈[0,i]
  • (j,dp[v][j])(j, dp[v][j])(j,dp[v][j]) 代表 vvv 这个物品组的容量和价值，这里的 iii 和 jjj 都是需要枚举的；
  • 由于 “如果选择一个物品，则必须选择它的父结点” 这个条件的存在，根结点必须占用 c[u]c[u]c[u] 的容量，所以剩余能给子结点分配的容量最多不能超过 m−c[u]m - c[u]m−c[u]；
  • 这个状态转移方程正是对应了分组背包的状态转移方程，只不过状态转移发生在树上，所以又可以把它称为树上分组背包问题。实际上，发生在树上的状态转移问题，我们也可以称之为 树形DP。但是树形 DP 不止树上分组背包，种类较多，关于树形 DP 的问题，作者会在未来的某个雷雨交加的夜晚，开辟另一章节来详细介绍。

4、代码分析

  • 实际求解过程还有一步后处理的过程，接下来我会给出一段代码，然后根据行后的注释对整个依赖背包的实现过程给出详细讲解。
void treeGroupKnapsack(int father, int u, int m) {
    int i, j, k;
    int maxCapacity = m - Knap[u].capacity;            // 1）
    for (i = 0; i <= maxCapacity; ++i)                 // 2）
        dp[u][i] = 0;
    for (int idx = 0; idx < edges[u].size(); ++idx) {  // 3）
        int v = edges[u][idx];
        if (v == father) continue;                     // 4）
        treeGroupKnapsack(u, v, maxCapacity);          // 5）
        for (i = maxCapacity; i >= 0; --i) {           // 6）
            for (j = 0; j <= i; ++j) {                 // 7）
                dp[u][i] = opt(
                    dp[u][i],
                    dp[u][i - j] + dp[v][j]);
            }
        }
    }
    for (i = m; i >= Knap[u].capacity; --i)            // 8）
        dp[u][i] = dp[u][i - Knap[u].capacity] + Knap[u].weight;
    for (i = 0; i < Knap[u].capacity; ++i)
        dp[u][i] = 0;
}
  • 1）基于这个问题父结点必选的情况，所以能够分配给子结点的最大容量为 m−c[u]m - c[u]m−c[u]，对应代码中的 maxCapacity = m - Knap[u].capacity。
  • 2）将给定容量范围内的 dp[u][i]dp[u][i]dp[u][i] 都初始化为 0，代表了目前一个子结点都不选的情况下的初始值，当然根结点 uuu 自己也不选。
  • 3）枚举所有子树，一棵子树对应分组背包的一个组。
  • 4）这一步是对无向树的一个容错，避免误把父结点当成是子结点造成循环递归。
  • 5）递归求解子结点所有组的状态，这个函数调用完毕以后 dp[v][0...m]dp[v][0 ... m]dp[v][0...m] 就都计算出来了。
  • 6）对应了分组背包的逆序枚举容量，必须逆序，不理解的读者可以参考前面章节的分组背包的降维思想。
  • 7）对容量为 jjj, 价值为 dp[v][j]dp[v][j]dp[v][j] 的物品进行决策，执行 最多选一个 的分组背包。
  • 8）这一步是最后的关键一步，由于组合完所有子树的状态后，计算出的 dp[u][i]dp[u][i]dp[u][i] 是没有包含根结点 uuu 的，而根结点是必须包含的，所以我们需要再进行一次状态转移，即：dp[u][i]=dp[u][i−c[u]]+w[u]dp[u][i] = dp[u][i-c[u]]+w[u]dp[u][i]=dp[u][i−c[u]]+w[u]
  • 代表了在 i−c[u]i-c[u]i−c[u] 这个背包里放入容量为 c[u]c[u]c[u]，价值为 w[u]w[u]w[u] 的物品，并且对于 i<c[u]i < c[u]i<c[u] 的 dp[u][i]dp[u][i]dp[u][i] 恒等于 0，含义是根结点都选不了，那么整个子树都无法选进背包。

五、时间复杂度分析

  • 一般依赖背包（树上分组背包）问题的状态总数为 O(nm)O(nm)O(nm)，状态转移的消耗为 O(m)O(m)O(m)，所以，整个算法的时间复杂度为 O(nm2)O(nm^2)O(nm2)。
  • 虽然这里是采用的递归，但是基于树形的特殊结构，每个结点只会被至多访问一次，每次访问完毕以后会进行一个 O(m2)O(m^2)O(m2) 的分组背包进行状态计算。所以时间复杂度并不是指数级的。
  • 以上就是一般依赖背包的所有内容了，接下来介绍几种隐式的依赖背包问题，题目本身没有背包和物品的概念，需要把问题抽象出来，虽然问法不同，但是模式是大同小异的。

四、依赖背包问题的变种

1、父结点选择则子结点不必选

  • 依赖背包的描述是：如果选择一个物品，则必须选择它的父结点；而这里要讨论的问题是：如果选了父结点，则子结点不能再选。看似不同的问题，却可以用相同的思路求解。

【例题3】有 n(n≤200)n(n \le 200)n(n≤200) 个国家，给出国家之间的统治关系，如果选择国家 aaa，那么所有被它直接或者间接统治的国家都能够被选进来，选择第 iii 个国家的消耗为 w[i]w[i]w[i]，问要选择至少 mmm 个国家的最少消耗。

  • 这个问题同样也是一个树形的依赖关系，当然，这里可能不止是一棵树，而是一个森林，我们可以把所有没有被统治的国家，都认为是被一个叫 “超级大国” 的国家统治，并且选择 “超级大国” 的消耗设定为 无限大，毕竟这个国家是不存在的，所以不能选它，这样就把问题转换成了只有一棵树的情况；
  • 令 dp[u][i]dp[u][i]dp[u][i] 表示以 uuu 这个国家为子树的根结点，能够选正好 iii 个国家的最少消耗，如果方案不存在，值为无穷大，实际编码过程可以定义宏 #define inf 1000000000，初始时 dp[u][0]=0dp[u][0] = 0dp[u][0]=0；
  • 对于每个国家 uuu，决策有两种：
  • 1）不选择自己，那么对每个子树的物品组 (v,dp[v][j])(v, dp[v][j])(v,dp[v][j]) j∈[0,i]j \in [0,i]j∈[0,i] 进行分组背包计算；
  • 2）选择自己，那么就可以认为是一个容量为 treesize[u]treesize[u]treesize[u]，价值为 w[u]w[u]w[u] 的物品，其中 treesize[u]treesize[u]treesize[u] 代表以 uuu 为子树的结点总数，可以通过深搜遍历完毕统计得出；dp[u][treesize[u]]=min(dp[u][treesize[u]],w[u])dp[u][ treesize[u] ] = min(dp[u][ treesize[u] ], w[u])dp[u][treesize[u]]=min(dp[u][treesize[u]],w[u])
  • 这个状态转移方程的含义是：对于当前国家， dp[u][treesize[u]]dp[u][ treesize[u] ]dp[u][treesize[u]] 代表选择子树的最少消耗，w[u]w[u]w[u]代表选择自己的消耗，两者取小者。
  • 最后取 dp[root][m...n]dp[root][m...n]dp[root][m...n] 中的最小值就是答案。

2、树的可重复访问最大权回路

【例题4】n(n≤100)n (n \le 100)n(n≤100) 个房间，房间之间连成一个树的形状，任意两个房间国家之间走动都会消耗时间。每个房间有一些宝物，现在想要从 1 号 房间到 nnn 号房间，给出最多只能消耗 t(t≤100)t (t \le 100)t(t≤100) 分钟，问能够带走最多的宝物价值。如果无法从 1 到 nnn，输出 -1（令任意两条边之间行走消耗 1 分钟）。

  • 考虑有的房间是会走两遍的，而 1→n1 \to n1→n 的这条路径，走两遍是没有意义的，但是也至少要走一遍，那么 1→n1 \to n1→n 这条路径的时间消耗免不了，所以如果 1→n1 \to n1→n 的最小耗时为 tmint_{min}tmin​，当 tmin>tt_{min} > ttmin​>t 则无法逃出；
  • 如果 tmin≤tt_{min} \le ttmin​≤t 时，我们可以令 t=t−tmint = t - t_{min}t=t−tmin​，并且可以把 1→n1 \to n1→n 的边上的时间消耗都标记为 0，因为其它路径都是走两遍，如果这里的路径标记为 0 以后，走 1 遍和走两遍消耗是一样的，这样就把问题统一起来了；
  • 令 dp[u][i]dp[u][i]dp[u][i] 表示从 uuu 出发，经过 uuu 的子树，在正好 iii 分钟且回来得到的最大价值；
  • 显然，初始情况下，dp[u][0]=w[u]dp[u][0] = w[u]dp[u][0]=w[u]，其它情况都为无穷小；
  • 因为给 uuu 分配的时间为 iii，所以对于 uuu 的每棵子树 vvv 最多可以使用的时间为 i−2i-2i−2，这里的 2 表示 (u,v)(u, v)(u,v)这条边来回走两次；子树看成一个物品组，为 (j,dp[v][j])(j, dp[v][j])(j,dp[v][j])，j∈[0,i−2]j \in [0, i-2]j∈[0,i−2]，得到分组背包的状态转移方程为：dp[u][i]=max(dp[u][i],dp[u][i−2−j]+dp[v][j])dp[u][i] = max(dp[u][i], dp[u][i-2-j] + dp[v][j])dp[u][i]=max(dp[u][i],dp[u][i−2−j]+dp[v][j])
  • 最后，递归求解即可。

3、树的可重复访问最大权路

【例题5】给定一棵 n(n≤100)n(n \le 100)n(n≤100) 个结点的树，iii 号结点权值为 w[i]w[i]w[i]，问从 1 号节点出发，经过最多 k(k≤200)k(k \le 200)k(k≤200) 步可以累计的最大权值。

  • 这题和【例题4】的区别是，每个点沿着子树往下走是可以不回来的，也就是不用构成回路。
  • 所以，对于任意一个结点为根的子树，可以选择下去了回来，也可以选择下去了不回来。
  • 令 dp[0][u][i]dp[0][u][i]dp[0][u][i] 表示从 uuu 出发，经过 uuu 的子树，在正好 iii 步时回来得到的最大价值；而 dp[1][u][i]dp[1][u][i]dp[1][u][i] 表示从 uuu 出发，经过 uuu 的子树，用 iii 步且不回到 uuu 的最大价值。对这两种情况分别进行分析：

1）回来

  • 显然，初始情况下，dp[0][u][0]=w[u]dp[0][u][0] = w[u]dp[0][u][0]=w[u]，其它情况都为无穷小；
  • 如图四-3-1所示，因为给 uuu 分配的步数为 iii，所以对于 uuu 的每棵子树 vvv 最多可以使用的步数为 i−2i-2i−2，这里的 2 表示 (u,v)(u, v)(u,v)这条边来回走两次；子树看成一个物品组，为 (j,dp[0][v][j])(j, dp[0][v][j])(j,dp[0][v][j])，j∈[0,i−2]j \in [0, i-2]j∈[0,i−2]，得到分组背包的状态转移方程为：dp[0][u][i]=max(dp[0][u][i],dp[0][u][i−2−j]+dp[0][v][j])dp[0][u][i] = max(dp[0][u][i], dp[0][u][i-2-j] + dp[0][v][j])dp[0][u][i]=max(dp[0][u][i],dp[0][u][i−2−j]+dp[0][v][j])
    图四-3-1
  • 这一步和【例题4】类似。

2）不回来

  • 不回来的情况相对较为复杂一点，首先，对于 uuu 为根的子树，dp[0][u][0]=w[u]dp[0][u][0] = w[u]dp[0][u][0]=w[u]，其它情况为无穷小。
  • 对于当前枚举的子树 vvv，有两种情况：从结点 vvv 出发不回来，从非 vvv的结点出发不回来；
  • 1）从结点 vvv 出发不回来：选择 vvv 出发走 jjj 步并且不回来，则获得的最大价值为 dp[1][v][j]dp[1][v][j]dp[1][v][j]，那么在这之前，它还可以选择其它结点走 i−1−ji-1-ji−1−j 步并且回来，即 dp[0][u][i−1−j]dp[0][u][i-1-j]dp[0][u][i−1−j]，两者相加为其中一类可行解：aij=dp[0][u][i−1−j]+dp[1][v][j]a_{ij} = dp[0][u][i-1-j] + dp[1][v][j]aij​=dp[0][u][i−1−j]+dp[1][v][j]

  • 2）从非 vvv 的结点出发不回来：选择 vvv 出发走 jjj 步并且回来，则获得的最大价值为 dp[0][v][j]dp[0][v][j]dp[0][v][j]，那么在这之后，它可以选择其它结点走 i−2−ji-2-ji−2−j 步并且不回来，即 dp[1][u][i−2−j]dp[1][u][i-2-j]dp[1][u][i−2−j]，两者相加为另一类可行解：bij=dp[0][v][j]+dp[1][u][i−2−j]b_{ij} = dp[0][v][j] + dp[1][u][i-2-j]bij​=dp[0][v][j]+dp[1][u][i−2−j]
