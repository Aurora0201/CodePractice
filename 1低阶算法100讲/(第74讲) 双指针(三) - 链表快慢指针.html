一、概念定义

  链表的快慢指针，就是定义两个指针，一个指针可以走得很快，另一个指针走的相对较慢，当快指针走到链表结尾，慢指针对应的结点，就能获取一些信息。从而辅助某些问题的求解。
  例如，想要求解链表的倒数第 kkk 个结点。假设链表总长度为 nnn，由于这个 nnn 是未知的，所以我们可以先定义一个快指针fast，先让它走k步，然后再定义一个慢指针slow。
  两个指针一起往前推进，当fast指针无路可走的时候，说明，慢指针slow走了n-k步，于是慢指针对应的那个结点就是倒数第k个结点。

二、题目描述

  给定一个链表，返回链表开始入环的第一个节点。 从链表的头节点开始沿着 next指针进入环的第一个节点为环的入口节点。如果链表无环，则返回 null。
  为了表示给定链表中的环，我们使用整数 pos来表示链表尾连接到链表中的位置（索引从 0 开始）。 如果 pos是 -1，则在该链表中没有环。注意，pos仅仅是用于标识环的情况，并不会作为参数传递到函数中。

三、算法详解

  (1)(1)(1) 假设从起点走到环的入口的步数为 xxx，链表环结点个数为 yyy，快慢指针表示一个指针走一步，一个指针走两步，会在距离环形入口的 zzz 处相遇；

  (2)(2)(2) 慢指针走了 nnn 圈花费的时间为：
x+n∗y+zx + n * y + zx+n∗y+z
  (3)(3)(3) 快指针走了 mmm 圈花费的时间为：
x+m∗y+z2\frac {x + m * y + z}{2}2x+m∗y+z​
  (4)(4)(4) 基于时间相等的原理，可得：
x+n∗y+z=x+m∗y+z2x + n * y + z = \frac {x + m * y + z}{2}x+n∗y+z=2x+m∗y+z​
  (5)(5)(5) 等式两边同时乘上 2，并且进行移项，得到：
2x+2z+2ny=x+z+myx+n∗y+z=(m−n)y2x + 2z + 2ny = x + z + my \\ x + n*y + z = (m - n)y 2x+2z+2ny=x+z+myx+n∗y+z=(m−n)y

  (6)(6)(6) 用 yyy 来表示 xxx，得到：
x=(m−2∗n−1)y+(y−z)x = (m-2*n - 1)y + (y - z)x=(m−2∗n−1)y+(y−z)
  (7)(7)(7) 这个公式的含义就是：快慢指针一起往前走，当他们相遇的时候，把快指针重新指向链表头结点。然后继续一起走，那么当慢指针再走 y−zy-zy−z 步，则快指针就会到入口节点，这时候它们又相遇啦！于是，只需要模拟这个过即可。

四、源码剖析

struct ListNode *detectCycle(struct ListNode *head) {
    struct ListNode *fast = head, *slow = head;
    if(fast == NULL) {
        return NULL;
    }
    while(1) {                   // (1)
        slow = slow->next;
        fast = fast->next;
        if(fast == NULL) {
            return NULL;
        }
        fast = fast->next;
        if(fast == NULL) {
            return NULL;
        }
        if(fast == slow) break;
    }
    fast = head;                 // (2)
    while(fast != slow) {        // (3)
        fast = fast->next;
        slow = slow->next;
    }
    return fast;
    
}
  • (1)(1)(1) 对于快慢指针，快指针每次走两步，慢指针每次走一步，相遇的时候跳出循环；
  • (2)(2)(2) 将快指针置到链表开始，继续快慢一起同时推进。
  • (3)(3)(3) 当两者相遇的时候，就一定是链表的入口节点；
