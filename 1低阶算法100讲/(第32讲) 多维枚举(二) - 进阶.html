一、概念定义

  在之前的章节中，我们已经学过了 线性枚举，就是利用一个循环来进行穷举和计算。那么，多维枚举就是多个循环之间的嵌套。格式如下：

int i, j, k;
for(i = 0; i < x; ++i) {
    for(j = 0; j < y; ++j) {
    	for(k = 0; k < z; ++k) {
        	....
        }
    }
}

  更加深入的理解一下，枚举的本质到底是什么？
  本质是把 变量 变成 常量。如上面的枚举中，第一层循环中的 iii 原本是一个变量，但是在枚举的过程中将它具象化，变成了常量。好处就是在循环体内，它可以自由使用，不需要考虑任何由变量带来的不确定因素（这句话如果不理解，就先不要看吧，我还没想好怎么解释比较贴切）。
  当然，实际的问题中，如何枚举，需要根据问题本身进行推敲。

二、题目描述

  给你一个 下标从 000 开始 的不超过 100100100 的正整数数组 nums ，返回满足下述条件的 不同 四元组 (a,b,c,d)(a, b, c, d)(a,b,c,d) 的 数目(a<b<c<d)(a < b< c< d)(a<b<c<d)：
nums[a]+nums[b]+nums[c]==nums[d]nums[a] + nums[b] + nums[c] == nums[d]nums[a]+nums[b]+nums[c]==nums[d]

三、算法详解

  比较简单的思路就是四层循环分别枚举 a,b,c,da,b,c,da,b,c,d，然后进行可行性判定。时间复杂度为 O(n4)O(n^4)O(n4)。
  我们可以将等式做个变换，如下：nums[a]+nums[b]==nums[d]−nums[c]nums[a] + nums[b] == nums[d] - nums[c]nums[a]+nums[b]==nums[d]−nums[c]
  先枚举 ccc，这样 nums[c]nums[c]nums[c] 就变成了常量，并且 a,ba,ba,b 的枚举可以和 ddd 的枚举分开。枚举 a,ba,ba,b 并且将 nums[a]+nums[b]nums[a]+nums[b]nums[a]+nums[b] 映射到哈希表中；然后就可以枚举 ddd 并且在哈希表中 O(1)O(1)O(1) 查找 nums[d]−nums[c]nums[d]-nums[c]nums[d]−nums[c]是否存在了。

四、源码剖析

1、暴力枚举

int countQuadruplets(int* nums, int n){
    int a, b, c, d;
    int sum = 0, x = 0;

    for(a = 0; a < n; ++a) {
        for(b = a+1; b < n; ++b) {
            for(c = b + 1; c < n; ++c) {
                x = nums[a] + nums[b] + nums[c];   // (1)
                if(x > 100) {
                    continue;                      // (2)
                }
                for(d = c+1; d < n; ++d) {
                    if(nums[d] == x) ++sum;        // (3)
                }
            }
        }
    }
    return sum;
}
  • (1)(1)(1) 三层循环枚举等式左边的部分；
  • (2)(2)(2) 如果数值超过范围，则直接跳过；
  • (3)(3)(3) 第四层循环枚举等式右边的部分；

  这样做的时间复杂度为 O(n4)O(n^4)O(n4)，我们可以利用简易的哈希数组进行降维；

2、哈希降维

int countQuadruplets(int* nums, int n){
    int a, b, c, d;
    int hash[303];
    int sum = 0;

    for(c = 0; c < n; ++c) {                      // (1)
        memset(hash, 0, sizeof(hash));           
        for(a = 0; a < c; ++a) {                
            for(b = a + 1; b < c; ++b) {
                hash[ nums[a] + nums[b] ]++;      // (2)
            }
        }
        for(d = c+1; d < n; ++d) {
            if( nums[d] - nums[c] >= 0 ) {
                sum += hash[ nums[d] - nums[c] ]; // (3)
            }
        }
    }
    return sum;
}
  • (1)(1)(1) 先枚举 ccc，将 ccc 变成常量；
  • (2)(2)(2) 然后可以枚举 aaa 和 bbb，由于 a<b<ca < b < ca<b<c，所以枚举的上界是 c−1c-1c−1，将枚举得到的结果 nums[a]+nums[b]nums[a] + nums[b]nums[a]+nums[b] 映射到哈希表中；
  • (3)(3)(3) 最后，再枚举 ddd，将 ddd 也变成常量，这样就可以快速计算出 nums[d]−nums[c]nums[d] - nums[c]nums[d]−nums[c]，然后再去哈希表中查找，累加到结果中即可；

  这样做，我们发现，循环的最大嵌套层次是三层，时间复杂度成功变为 O(n3)O(n^3)O(n3)，降了一维。
