一、概念定义

  贪心问题让人又爱又恨，因为贪心问题没有固定的解法，相比动态规划而言，贪心问题要难起来，会更难。
  贪心问题的难点在于 「 证明 」，也就是说，你怎么知道这种做法是对的，有时候可能只是想当然。
  本文会介绍一种一眼就能看出来的贪心，一般就是对于某个数组，进行一次递增或者递减排序以后，然后按照题目模拟做一遍即可。

二、题目描述

  给定一个 n(n≤1000)n(n \le 1000)n(n≤1000) 个元素的包含非负整数的数组，你的任务是统计其中可以组成三角形三条边的三元组个数。

三、算法详解

  (1)(1)(1) 首先，将所有数按照递增排序；
  (2)(2)(2) 对于三条边 iii、jjj、kkk 的下标满足 i<j<ki \lt j \lt ki<j<k，如果nums[i] + nums[j] <= nums[k]，则不可能组成三角形，直接可以跳出循环，不需要再枚举 kkk；
  (3)(3)(3) 于是，枚举 iii，在 iii 固定的情况下，满足条件的边数就是 (k,j)(k,j)(k,j) 二元组的对数；

四、源码剖析

int cmp(const void *a, const void *b) {
    return *(int *)a - *(int *)b;
}

int triangleNumber(int* nums, int numsSize){
    int i, j, k;
    int ans = 0;
    qsort(nums, numsSize, sizeof(int), cmp);       // (1)
    for(i = 0; i < numsSize; ++i) {
        j = i + 1;
        k = j + 1;                                 // (2)
        while(j < numsSize) {
            while(k < numsSize) {
                if(nums[i] + nums[j] <= nums[k]) {
                    break;                         // (3)
                }
                ++k;               
            }
            ans += k-j-1;                          // (4)
            ++j;                                   // (5)
            if(k == j) k++;
        }
        
    }    
    return ans;
}
  • (1)(1)(1) 首先，将所有数组元素按照递增排序；
  • (2)(2)(2) 利用三个游标 iii、jjj、kkk，递增枚举三条边；
  • (3)(3)(3) 如果遇到nums[i] + nums[j] <= nums[k]则直接跳出第三层循环，因为边是递增的，后面的 kkk 都会满足这个条件，从而无法构成三角形；
  • (4)(4)(4) 在 iii 固定的情况下，满足条件的边数就是 (k,j)(k,j)(k,j) 二元组的对数；
  • (5)(5)(5) 继续自增 jjj，如果发现 jjj 和 kkk 相等，则自增 kkk；
    总体来说，就是枚举 iii 和 jjj，kkk 是跟着 jjj 一起走的，所以时间复杂度为 O(n2)O(n^2)O(n2)。
